什么是强密码？

强密码应具备以下特征：

  • 长度至少 12 位（推荐 16 位以上）
  • 混合多种字符：大小写字母 + 数字 + 特殊符号
  • 无规律可循：避免字典单词、生日、姓名
  • 独一无二：每个账户使用不同密码
☠️

毒密码的特征

  • 短于 8 位
  • 使用连续数字（123456）或重复字符（aaaa）
  • 使用常见单词（password、admin）
  • 包含个人信息（生日、电话）
  • 所有网站使用同一密码
  • 从不更换密码
🛡️

如何安全存储密码？

  • 使用密码管理器（如 1Password、Bitwarden）
  • 开启双重验证 (2FA)
  • 定期更换重要密码
  • 不要将密码写在纸上或存储在纯文本中
  • 使用生物识别替代简单密码
🎯

密码攻击手段

  • 暴力破解：尝试所有可能的组合
  • 字典攻击：使用常见密码字典
  • 彩虹表：预计算的哈希值反查
  • 钓鱼攻击：伪造登录页面骗取密码
  • 社会工程学：利用人性弱点获取密码
✅

密码安全最佳实践

  • 使用密码短语（如：Coffee$Sunset@2026!）
  • 启用密码管理器生成随机密码
  • 为重要账户开启2FA 验证
  • 定期检查账户是否存在数据泄露
  • 使用无密码登录（Passkeys）新技术

📖 Passkey 是什么？

Passkey 是基于 FIDO2/WebAuthn 标准的无密码认证方式。

🔑 传统登录：你 → 输入密码 → 服务器验证密码哈希

🔑 Passkey 登录：你 → 指纹/面容 → 设备用私钥签名 → 服务器用公钥验证

✅ 私钥永远不离开你的设备

✅ 服务器只存储公钥（泄露也无用）

✅ 绑定域名，无法被钓鱼

✅ 不需要记密码，不需要 2FA

🔄 注册流程详解

1. 网站向浏览器发起注册请求（navigator.credentials.create） 2. 浏览器弹出系统级认证弹窗（指纹/面容/Windows Hello） 3. 用户验证成功 → 设备生成一对公钥/私钥 4. 私钥安全存储在设备的 Secure Enclave / TPM 芯片中 5. 公钥 + 凭证 ID 发送回网站服务器保存 6. 注册完成！

🔄 登录流程详解

1. 网站向浏览器发起登录请求（navigator.credentials.get） 2. 浏览器弹出系统级认证弹窗 3. 用户验证成功 → 设备用私钥对 challenge 签名 4. 签名结果发送回网站服务器 5. 服务器用存储的公钥验证签名 6. 验证通过 → 登录成功！
